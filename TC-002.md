# Duplicate Message Handling Test

## Test Logic — `testDuplicateMessageHandling`

### 🌱 Seed Data
Sets up a unique product **QA-DUPLICATE** with **100 stock**.

---

### 🧪 Test Steps

| Step | Description |
|------|-------------|
| **Step 1** — API Order | Submits a normal order through the Producer API |
| **Step 2** — Capture Payload | Uses the `KafkaVerifier` to capture the exact JSON payload published to Kafka |
| **Step 3** — Simulate Retry | Uses the new `KafkaProducerClient` to publish the identical JSON payload a second time |
| **Step 4** — Database Assertion | Verifies that only **one record** exists in MongoDB for that `orderId` |
| **Step 5** — Stock Assertion | Verifies that the stock was only decremented once *(remaining stock: **98**)* |

---

### ✅ Success Criteria

- ✅ Only **one record** exists in MongoDB for the duplicate order
- ✅ Product stock is only **decremented once**
- ✅ The consumer correctly identifies the duplicate via `_id` and commits the offset **without double-processing**
